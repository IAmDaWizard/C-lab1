## Описание лабораторной работы

В ходе выполнения лабораторной работы была реализована программа для работы с ориентированным графом, представленным с помощью матрицы смежности.

Граф хранится в структуре `matrix`, содержащей:

- двумерный массив `vertices`, отвечающий за матрицу смежности;
- массив `numbers`, содержащий значения вершин;
- поле `size`, хранящее количество вершин графа.

В матрице смежности используется следующая логика:

```text
vertices[i][j] = 1
```

означает, что в графе существует ориентированное ребро:

```text
i -> j
```

То есть строка матрицы показывает, из какой вершины выходит ребро, а столбец — в какую вершину оно входит.

## Описание используемого представления графа

Для хранения графа используется матрица смежности.

Если в графе есть вершины:

```text
1 2 3
```

и рёбра:

```text
1 -> 2
1 -> 3
2 -> 3
```

то матрица смежности имеет вид:

```text
      1  2  3
1     0  1  1
2     0  0  1
3     0  0  0
```

Каждый элемент матрицы показывает наличие или отсутствие ребра:

```text
0 — ребра нет
1 — ребро есть
```

Так как для топологической сортировки требуется ориентированный ациклический граф, рёбра добавляются с направлением.

Например:

```text
1 -> 2
```

означает, что вершина `1` должна идти раньше вершины `2` в результате топологической сортировки.

## Описание алгоритма топологической сортировки

Топологическая сортировка — это упорядочивание вершин ориентированного ациклического графа таким образом, что для каждого ребра:

```text
u -> v
```

вершина `u` находится раньше вершины `v`.

Для реализации был использован алгоритм Кана.

### Основная идея алгоритма

1. Для каждой вершины считается количество входящих рёбер.
2. Находится вершина, у которой количество входящих рёбер равно нулю.
3. Такая вершина добавляется в результат.
4. Все исходящие из неё рёбра мысленно удаляются.
5. Для вершин, в которые вели эти рёбра, уменьшается количество входящих рёбер.
6. Процесс повторяется до тех пор, пока возможно находить вершины с нулевой входящей степенью.
7. Если были обработаны все вершины, топологическая сортировка успешно завершена.
8. Если обработаны не все вершины, значит в графе есть цикл, и топологическая сортировка невозможна.

---

## Пример работы алгоритма

Пусть задан граф:

```text
1 -> 3
2 -> 3
3 -> 4
```

Входящие степени вершин:

```text
1: 0
2: 0
3: 2
4: 1
```

Сначала можно взять вершину `1`, так как в неё не входит ни одного ребра.

После удаления ребра:

```text
1 -> 3
```

входящая степень вершины `3` уменьшится:

```text
3: 1
```

Затем берётся вершина `2`.

После удаления ребра:

```text
2 -> 3
```

входящая степень вершины `3` становится равной нулю.

После этого берётся вершина `3`, затем вершина `4`.

Один из возможных результатов топологической сортировки:

```text
1 2 3 4
```

# Benchmarking

Для оценки производительности программы был использован Google Benchmark.

Google Benchmark позволяет запускать функцию много раз и получать усреднённое время выполнения. В benchmark измерялись две основные операции:

1. Удаление вершины из графа.
2. Топологическая сортировка ориентированного ациклического графа.

Печать графа не измерялась, так как операции вывода в консоль зависят от скорости ввода-вывода и не отражают скорость работы алгоритма.

## Условия проведения benchmark

Benchmark проводился на ориентированных ациклических графах разных размеров:

```text
10, 50, 100, 300, 500 вершин
```

Граф генерировался как DAG — ориентированный ациклический граф. Рёбра добавлялись только из вершины с меньшим индексом в вершину с большим индексом. Это гарантирует отсутствие циклов.

Для каждого размера графа измерялись:

- `BM_DeleteVertex` — удаление вершины из графа;
- `BM_TopologicalSort` — выполнение топологической сортировки.

## Анализ результатов benchmark

По результатам benchmark видно, что время выполнения операций увеличивается с ростом количества вершин графа.

Удаление вершины выполняется быстрее, чем топологическая сортировка, так как при удалении требуется сдвиг строк и столбцов матрицы смежности, а также обновление массива вершин.

Топологическая сортировка работает медленнее, потому что алгоритм дополнительно считает входящие степени вершин и многократно проходит по матрице смежности.

Так как граф представлен матрицей смежности, многие операции требуют прохода по строкам и столбцам матрицы. Поэтому при увеличении количества вершин время выполнения растёт достаточно заметно.

## Сложность алгоритмов

### Добавление вершины

При добавлении вершины происходит расширение матрицы смежности и добавление новой строки и нового столбца.

Сложность:

```text
O(n)
```

где `n` — количество вершин графа.

---

### Удаление вершины

При удалении вершины выполняются:

- удаление значения вершины из массива `numbers`;
- сдвиг строк матрицы;
- сдвиг столбцов матрицы;
- уменьшение количества вершин.

Сложность:

```text
O(n^2)
```

так как необходимо обработать элементы матрицы смежности.

---

### Топологическая сортировка

Топологическая сортировка через матрицу смежности требует прохода по матрице для подсчёта входящих степеней, а затем обработки исходящих рёбер.

Сложность:

```text
O(n^2)
```

где `n` — количество вершин графа.
